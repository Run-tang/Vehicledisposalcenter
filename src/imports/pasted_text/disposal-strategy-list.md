处置策略列表需求详情
模块	功能说明
1. 导航与入口	【字段说明】：
一级模块名称：【处置中心】；
二级目录名称：【处置策略】、【批发处置记录】、【零售处置记录】。
【功能说明】：定义“广联二手车”管理后台的目录树结构，明确处置中心的物理位置，位于【库存管理】之后，【检测管理】之前。
【交互说明】：左侧目录树导航，点击二级目录进入对应的功能页面。
2. 顶部状态切换 (Tabs)	【字段说明】：包含【全部】、【待处置】、【执行中】、【已完成】、【已取消】。
【功能说明】：按照策略的生命周期状态进行分类展示。标题格式为“状态名称(数量)”。
【交互说明】：
1. 点击不同Tab，下方列表数据实时过滤。
2. Tab中的数量需与当前全局搜索器（筛选区域）的条件关联，搜索后数量需同步更新。
3. 筛选区域 (Filter)	【字段说明】：
1. VIN码：用于车辆唯一标识搜索，支持模糊匹配。
2. 车牌号：支持模糊匹配。
3. 所属门店：枚举类型，来源于机构权限配置。
4. 当前处置平台：枚举类型（优加车拍、有辆、汽车街、优信拍等）。
5.策略状态：枚举类型。
6. 处置方式：批发、零售。
7. 零售车辆类型：文本模糊搜索。
8.批发车辆类型：枚举筛选
【功能说明】：提供多维度数据检索能力，以便运营人员快速定位车辆。
【交互说明】：
1. 提供【筛选】按钮执行搜索，提供【重置】按钮清空所有输入项。
2. 筛选数据与表头字段规则保持一致。
4. 操作区域 (Batch Action)	【字段说明】：【导出数据】按钮。
【功能说明】：支持数据的本地化保存。
【交互说明】：
1. 导出逻辑：若用户通过复选框勾选了部分数据，点击导出则仅导出选中项；若未勾选，默认导出当前筛选条件下的全部数据。
2. 导出表头增强：除了列表可见表头外，导出的Excel需包含【经销差价】字段。计算公式为：成交价 - 收购价。
增加：批发-车况信息特别描述
增加：取消原因（仅针对状态为已取消的行数据，展示其备注内容）。
增加：批发-保留价异常说明：创建时存在的保留价低于标准保留价展示，无
增加：零售-保留价异常说明 ：创建时存在的寄售底价低于标准保留价展示，无则为-
增加：市场参考价格区间：根据当前市场行情，该车辆的预估销售价格范围。来自新增策略。

5. 策略列表 - 基础信息	【字段说明】：
1. 复选框：用于勾选。 
2. 策略ID：系统自动生成的唯一流水号。
3. 车牌号：车辆档案中的车牌。
4. 品牌车型：展示格式为“品牌-车型名称”。
5. 车辆所在地：展示格式为“省-市”。
6. 收车时间：年月日，指收车审批完成的日期。
7. 收购价：单位“万元”，保留两位小数。
【交互说明】：【品牌车型】字段若内容过长，需支持在单元格内以省略号显示，鼠标Hover时浮窗弹出完整名称。
6. 策略列表 - 处置信息	【字段说明】：
1. 车辆类型：来源根据数据字典，包含置换车、报废车、试驾车。若处置方式为零售，该字段展示为“-”。
2. 库存时长：单位“天”。计算逻辑：当前日期 - 收车日期。
3. 处置方式：展示为“批发”或“零售”。
4. 处置渠道：
- 若为“批发”：展示对应拍卖平台名称（优加、有辆等）。
- 若为“零售”：展示创建策略时手动填写的渠道名称。
5. 所属门店：收车时所归属的门店全称。
【功能说明】：核心展示区域，明确业务进展和资源走向。
7. 策略列表 - 财务指标	【字段说明】：
1. 当前保留价/底价：
  a. 若为“批发”：展示“批-xxx.xx万元”。
  b. 若为“零售”：展示“零-xxx.xx万元”。
  c. 当处置方式为「零售」时，展示以下字段：零售价格（取策略最近一次更新零售进度时的零售价格）、寄售底价（已有「当前保留价/底价」列覆盖）
2. 公司标准保留价：系统后台算法生成的兜底价格，单位“万元”。
3. 市场价格区间：由最起价和最高价组成的区间。
4. 拍卖最高价：零售展示为“-”；批发场景回填各平台最新的出价。无数据展示“-”。
5.零售价格：批发时展示为“-”；零售场景回填最新零售价。无数据展示“-”。
【交互说明】：数据需能实时根据外部平台回传的回调接口进行更新更新。
8. 策略列表 - 状态与进度	【字段说明】：
1. 策略状态：待处置、执行中、已完成、已取消。
待处置：车辆成功收上来后，展示的是待处置，每一个车都会有一个待处置的记录。这个时候没完成策略的制定，即初次策略未完成制定
执行中：创建策略策略后，策略状态就是执行中，这个时候会需要执行人员进行策略的执行与进度的更新。
已完成：当这个车成功卖出去（生成了订单）的情况下，策略的状态就已完成，车辆状态列应展示为"成交“”
已取消：取消了这个策略，这个车没卖出去会回到“待处置”中，同时已取消会记录历史的策略信息-快照模式。生成新的待处置的策略单。
2. 车辆状态：
- 内部平台（优加）：映射车辆档案实时车辆状态。
- 外部平台（批发）：待拍卖（创建策略后默认）、检测待上拍、流拍、仲裁、成交。
- 零售场景：寄售中、已成交、已取消。
3. 处置次数：正整数，单位“次”。逻辑：统计该VIN码下关联的所有处置记录条数。
4. 创建人/更新时间：记录策略的责任人及最近操作时间。
9. 策略操作 - 批发场景	【功能说明】：根据状态动态展示按钮。
1. 待处置：
新增策略：唤起新增策略弹窗
查看车辆档案：打开新的标签页，跳转车辆档案页
2. 执行中：
查看详情：点击后进入处置策略详情页。
编辑处置策略：唤起编辑策略弹窗，带出原有填写数据，处置方式可切换"。仅可修改除车辆以外的信息。
更新拍卖进度：点击后打开更新拍卖进度弹窗
取消策略：内部平台若已关联拍场则不允许取消（需先在拍场模块下架）；外部平台允许取消，需弹窗填写取消原因，并记录在流转日志中。
弹窗定义：取消原因文本框（必填，最大200字符）、确认/取消按钮
3. 已完成：
查看详情：点击后进入处置策略详情页。
查看拍卖订单：点击跳转拍卖订单详情页
4. 已取消：
展示“查看详情”、“查看取消原因”（点击在Tip浮窗中展示）。
【交互说明】：
若为“内部优加车拍”，不可手动更新进度，需自动流转。将【更新拍卖进度】置灰并提示"内部平台动流转"
- 若为“外部平台”，点击“更新拍卖进度”唤起弹窗，填写最高价、流拍原因、拍卖平台反馈。若更新为“成交”，策略同步变更为“已完成”。
10. 策略操作 - 零售场景	【功能说明】：针对零售链路的操作维护。
【交互说明】：
1. 待处置：
新增策略：唤起新增策略弹窗
查看车辆档案：打开新的标签页，跳转车辆档案页
2. 执行中：
编辑处置策略：唤起编辑策略弹窗，带出原有填写数据，处置方式可切换"。仅可修改除车辆以外的信息。
查看详情：点击后进入处置策略详情页。
更新零售进度：打开"更新记录弹窗-零售"
3. 已完成：
查看详情：跳转策略详情页
查看零售订单：跳转关联订单详情页
4. 已取消
查看详情：跳转策略详情页

新增策略弹窗需求详情 
模块	功能说明（包含字段说明、功能说明、交互说明）
1. 策略概览 	-   1.1 策略ID
-   字段说明：策略的唯一内部标识符。
-   功能说明：系统层面用于识别、追踪和管理每一个处置策略。用户无需手动填写。
-   交互说明：
-   展示方式：文本展示，如“STR20231027001”。
-   默认值：创建策略前为空，策略成功创建后由后端自动生成并赋值。
-   交互：不可编辑。

 -   1.2 选择需要处置的车辆
-   字段说明：指定本次处置策略所针对的库存车辆。
-   功能说明：核心步骤，确保策略与具体车辆的绑定。系统需确保一辆车在同一时间只能有一条“执行中”的处置策略。
-   交互说明：
-   交互状态：未选择
-   展示：区域内显示“请选择处置车辆”的提示语。
-   交互：点击此区域（或旁边的“选择车辆”按钮），唤起“车辆选择器”弹窗。
-   车辆选择器：现有车辆选择器
-   应展示当前处于“待处置”状态的库存车辆列表。
-   用户可单选一辆车。选择后，弹窗/侧边栏关闭，所选车辆的简要信息将回填至“新增策略”弹窗。
-   交互状态：已选择
-   展示：展示所选处置车辆的简要信息卡片。
该卡片包含以下字段：
车辆所在地、车辆保管人电话、库存时长、车架号、品牌、车型名称、收购价、车牌号、车身颜色、车辆保管人姓名、省市门店全称、所属门店、收车时间（详细字段说明见下方2. 车辆简要信息详情）。
-   交互：
-   修改车辆：提供“重新选择”按钮，点击后可再次唤起车辆选择器，允许用户更换已选车辆。
-   查看评估单详情：提供“在侧边栏查看收车-评估单详情”的链按钮。点击后，系统应打开一个新的侧边栏，展示当前车辆的详细收车评估单内容，方便运营人员核对车辆历史信息。
2. 车辆简要信息详情（已选择车辆后展示） 	-   2.1 车辆所在地
-   字段说明：车辆当前实际存放的城市。
-   功能说明：指示车辆的地理位置，影响后续物流、过户及买家地域性匹配。
-   交互说明：
-   展示方式：文本框。
-   默认值：默认映射所选车辆的库存地址。
-   交互：可修改。，用户可选择实际车辆所在地，修改后展示为实际车辆所在地。

 -   2.2 车辆保管人电话
-   字段说明：负责保管车辆的联系人电话。
-   功能说明：便于买家或平台内部人员在询价、看车、交车等环节联系车辆保管方。
手机号11位数字
-   交互说明：
-   展示方式：文本输入框。
-   默认值：默认映射所选车辆收车评估单中的保管人电话。
-   非必填：允许为空。
-   业务逻辑：若为平台自有车源为内部员工电话；若为外部合作车商车源，则展示外部联系电话。

 -   2.3 库存时长
-   字段说明：车辆从入库到当前策略创建时所经历的时间长度。
-   功能说明：作为评估车辆库存健康度、决策处置紧迫性的重要指标。
-   交互说明：
-   展示方式：文本展示，格式为“X天”。
-   计算规则：根据“收车时间”与当前系统时间计算得出（当前日期 - 收车日期）。
-   交互：不可编辑。

 -   2.4 车架号
-   字段说明：车辆的唯一识别代码（VIN码）。
-   功能说明：车辆的核心身份信息，用于识别和匹配车辆。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的VIN码。
-   交互：不可编辑。

 -   2.5 品牌
-   字段说明：车辆的一级品牌名称，例如“宝马”、“奔驰”。
-   功能说明：车辆的基础分类属性。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的品牌信息。
-   交互：不可编辑。

 -   2.6 车型名称
-   字段说明：车辆的四级车型名称，例如“S级 2020款 S 450 L”。
-   功能说明：车辆的具体型号信息，是定价和销售的重要依据。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的车型名称。
-   交互：不可编辑。

-   2.7 收购价
-   字段说明：平台收购此车辆时的成本价格。
-   功能说明：内部成本核算、盈亏分析以及保留价设定的重要参考。
-   交互说明：
-   展示方式：文本展示，单位“万元”，精确到两位小数。
-   默认值：映射所选车辆的收购价。
-   交互：不可编辑。

 -   2.8 车牌号

-   功能说明：车辆的另一个重要识别信息。
-   交互说明：
-   展示方式：文本展示。

-   交互：不可编辑。

 -   2.9 车身颜色
-   字段说明：车辆外观的颜色。
-   功能说明：车辆的外观特征，对买家有一定吸引力。
-   交互说明：
-   展示方式：文本展示。

-   功能说明：识别车辆的实际保管主体，方便职责追溯和沟通。
