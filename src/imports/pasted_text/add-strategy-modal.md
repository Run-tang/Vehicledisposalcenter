模块：新增策略弹窗
模块	功能说明（包含字段说明、功能说明、交互说明）
1. 策略概览 	-   1.1 策略ID
-   字段说明：策略的唯一内部标识符。
-   功能说明：系统层面用于识别、追踪和管理每一个处置策略。用户无需手动填写。
-   交互说明：
-   展示方式：文本展示，如“STR20231027001”。
-   默认值：创建策略前为空，策略成功创建后由后端自动生成并赋值。
-   交互：不可编辑。

 -   1.2 选择需要处置的车辆
-   字段说明：指定本次处置策略所针对的库存车辆。
-   功能说明：核心步骤，确保策略与具体车辆的绑定。系统需确保一辆车在同一时间只能有一条“执行中”的处置策略。
-   交互说明：
-   交互状态：未选择
-   展示：区域内显示“请选择处置车辆”的提示语。
-   交互：点击此区域（或旁边的“选择车辆”按钮），唤起“车辆选择器”弹窗。
-   车辆选择器：现有车辆选择器
-   应展示当前处于“待处置”状态的库存车辆列表。
-   用户可单选一辆车。选择后，弹窗/侧边栏关闭，所选车辆的简要信息将回填至“新增策略”弹窗。
-   交互状态：已选择
-   展示：展示所选处置车辆的简要信息卡片。
该卡片包含以下字段：
车辆所在城市、车辆保管人电话、库存时长、车架号、品牌、车型名称、收购价格、车牌号、车身颜色、车辆保管人姓名、省市门店全称、所属门店、入库时间（详细字段说明见下方2. 车辆简要信息详情）。
-   交互：
-   修改车辆：提供“重新选择”按钮，点击后可再次唤起车辆选择器，允许用户更换已选车辆。
-   查看评估单详情：提供“在侧边栏查看收车-评估单详情”的链按钮。点击后，系统应打开一个新的侧边栏，展示当前车辆的详细收车评估单内容，方便运营人员核对车辆历史信息。
2. 车辆简要信息详情（已选择车辆后展示） 	-   2.1 车辆所在城市
-   字段说明：车辆当前实际存放的城市。
-   功能说明：指示车辆的地理位置，影响后续物流、过户及买家地域性匹配。
-   交互说明：
-   展示方式：文本框/下拉选择框。
-   默认值：默认映射所选车辆的库存地址。
-   交互：可修改。点击后唤起城市选择器（支持省-市两级联动），用户可选择实际车辆所在城市，修改后展示为实际车辆所在地。

 -   2.2 车辆保管人电话
-   字段说明：负责保管车辆的联系人电话。
-   功能说明：便于买家或平台内部人员在询价、看车、交车等环节联系车辆保管方。
-   交互说明：
-   展示方式：文本输入框。
-   默认值：默认映射所选车辆收车评估单中的保管人电话。
-   非必填：允许为空。
-   业务逻辑：若为平台自有车源，通常为内部员工电话；若为外部合作车商车源，则展示外部联系电话。

 -   2.3 库存时长
-   字段说明：车辆从入库到当前策略创建时所经历的时间长度。
-   功能说明：作为评估车辆库存健康度、决策处置紧迫性的重要指标。
-   交互说明：
-   展示方式：文本展示，格式为“X天”。
-   计算规则：根据“入库时间”与当前系统时间计算得出（当前日期 - 入库日期）。
-   交互：不可编辑。

 -   2.4 车架号
-   字段说明：车辆的唯一识别代码（VIN码）。
-   功能说明：车辆的核心身份信息，用于识别和匹配车辆。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的VIN码。
-   交互：不可编辑。

 -   2.5 品牌
-   字段说明：车辆的一级品牌名称，例如“宝马”、“奔驰”。
-   功能说明：车辆的基础分类属性。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的品牌信息。
-   交互：不可编辑。

 -   2.6 车型名称
-   字段说明：车辆的四级车型名称，例如“S级 2020款 S 450 L”。
-   功能说明：车辆的具体型号信息，是定价和销售的重要依据。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的车型名称。
-   交互：不可编辑。

 -   2.7 收车价
-   字段说明：平台收购此车辆时的成本价格。
-   功能说明：内部成本核算、盈亏分析以及保留价设定的重要参考。
-   交互说明：
-   展示方式：文本展示，单位“万元”，精确到两位小数。
-   默认值：映射所选车辆的收购价格。
-   交互：不可编辑。

 -   2.8 车牌号
-   字段说明：车辆档案中记录的车牌号码。
-   功能说明：车辆的另一个重要识别信息。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的车牌号。
-   交互：不可编辑。

 -   2.9 车身颜色
-   字段说明：车辆外观的颜色。
-   功能说明：车辆的外观特征，对买家有一定吸引力。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的车身颜色。
-   交互：不可编辑。

 -   2.10 车辆保管人姓名
-   字段说明：负责保管车辆人员的姓名或所属合作平台名称。
-   功能说明：识别车辆的实际保管主体，方便职责追溯和沟通。
-   交互说明：
-   展示方式：文本输入框。
-   默认值：默认映射车辆收车评估单中的保管人姓名。
-   业务逻辑：
-   如果是平台内部自有车辆，应展示内部员工姓名，可手动输入。
-   如果是外部合作车源，则展示外部合作平台的名称或联系人姓名，可手动输入。
-   交互：可修改，手动输入外部平台名称/联系人姓名/内部员工姓名。

 -   2.11 省市门店全称
-   字段说明：车辆所属的完整门店名称，包含其所在省市信息。
-   功能说明：明确车辆的隶属关系及详细的物理位置。与“所属门店”区别在于，此为更详细的全称。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的所属门店全称。
-   交互：不可编辑。

 -   2.12 所属门店
-   字段说明：车辆实际归属的门店名称。
-   功能说明：车辆的运营或销售主体。
-   交互说明：
-   展示方式：文本展示。
-   默认值：映射所选车辆的所属门店名称。
-   交互：不可编辑。

 -   2.13 入库时间
-   字段说明：车辆进入平台库存管理系统的时间。
-   功能说明：计算库存时长、评估车辆价值变动、确定处置时机的重要时间戳。
-   交互说明：
-   展示方式：文本展示，格式为“年-月-日”。
-   默认值：映射所选车辆的入库时间。
-   交互：不可编辑。
3. 选择零售或批发 	-   字段说明：定义本次车辆处置的渠道类型，决定后续参数配置和流程走向。
-   功能说明：根据市场策略和车辆特性，选择B2B批发（面向车商拍卖）或B2C零售（面向个人买家）两种主要处置路径。
-   交互说明：
-   展示方式：单选按钮组，选项为“批发”、“零售”。
-   默认选中：默认选中“批发”。
-   交互逻辑：当用户选择“批发”时，下方表单区域动态显示“批发策略资料”相关字段；当用户选择“零售”时，动态显示“零售策略资料”相关字段。两者独占显示。
3.1 批发策略资料 (当选择“批发”时展示) 	-   3.1.1 选择车辆类型
-   字段说明：在批发场景下，对车辆的特定业务分类。
-   功能说明：用于区分不同来源、不同性质的车辆，便于对批发车源进行精细化管理和拍卖策略制定。
-   交互说明：
-   展示方式：单选按钮组（根据后台配置，在数据字典获取）。
-   可选值：置换车、报废车、试驾车、厂家资源车。
-   默认值：默认选择“置换车”。


 -   3.1.2 市场参考价格区间
-   字段说明：根据当前市场行情，该车辆的预估销售价格范围。
-   功能说明：为平台运营人员设定建议保留价提供市场依据，也为潜在买家提供参考价值，评估车辆的市场竞争力。
-   交互说明：
-   展示方式：两个数字输入框，中间用“-”连接，单位“万元”。例如：[10.00] - [12.50] 万元。
-   输入格式：最起价 [输入框] - 最高价 [输入框] 万元。
-   数据校验：
-   必填。
-   必须为正数，支持最多两位小数。
-   逻辑校验：最起价 ≤ 最高价（闭区间，允许相等）。若不符合，提示“开始价格不能大于结束价格”。
-   显示文案：提示“市场价格参考区间：最起价-最高价，单位：万元。闭区间，允许相等”。

 -   3.1.3 公司标准保留价
-   字段说明：根据公司财务模型和风险控制政策，该车辆最低可接受的出售价格。
-   功能说明：平台方内部的成本保护底线，是衡量策略盈利能力和风险的关键指标。
-   交互说明：
-   展示方式：文本展示，单位“万元”，精确到两位小数。
-   计算规则：由后台根据预设的计算公式和配置（收车车价进行加价：分段增加金额，1万以下加500、1-3万加800、3万以上1000）自动得出。
-   交互：不可编辑，作为业务参考值。

 -   3.1.4 填写过户时效要求
-   字段说明：对买家完成车辆过户手续的时间限制和相关说明。
-   功能说明：明确交易后买家的责任期限，避免车辆长期挂在平台名下，影响库存周转、资金占用和潜在法律风险。
-   交互说明：
-   展示方式：数字输入框 + 文本输入框。
-   输入项1：过户天数 (正整数)。
-   数据校验：必填，必须为正整数，最高可填写9999。
-   单位：输入框后自动带出“天”。
-   输入项2：过户时效说明 (文本框)。
-   功能：用于补充说明具体的过户要求，例如“节假日顺延”、“逾期将产生违约金（每日XX元）”等。
-   字符限制：最多200个字符。非必填。
-   显示文案：提示“填写多少天内过户。正整数：单位天，最多填写9999”。

 -   3.1.5 填写建议保留价
-   字段说明：运营人员根据市场实际情况、车辆特性和公司标准保留价给出的建议拍卖底价。
-   功能说明：设定一个具有市场竞争力的拍卖起始价，目标是促成交易，同时尽量保障公司利润。该值可调整，但需遵循一定规则。
-   交互说明：
-   展示方式：数字输入框，单位“万元”，精确到两位小数。
-   必填：输入框后方带“*”标识。
-   数据校验：
-   必须为正数，支持最多两位小数。
-   逻辑校验：建议保留价原则上应 ≥ 公司标准保留价。
-   若低于公司标准保留价，需补充字段 2 进行说明
-   输入项2：保留价异常说明 (文本框)。
-   功能：用于补充说明为什么保留价低于公司标准保留价。
-   字符限制：最多200个字符。当低于公司标准保留价时必填。
-   显示文案：提示“请具体说明建议保留价低于公司保留价的理由”。
-   交互文案：提示“建议保留价，单位：万元”。

 -   3.1.6 车况信息特别描述
-   字段说明：对车辆收车评估报告中未提及或有变动的特殊车况进行补充说明。
-   功能说明：最大化信息透明度，确保买家在拍卖前充分了解车辆的真实状况，减少交易纠纷和退货风险。自动带出收车填写的车况说明（来自车辆档案）。
-   交互说明：
-   展示方式：多行文本输入框。非必填，
-   字符限制：最多500个字符。
-   业务逻辑：主要用于补充说明与收车评估单不一致的细节，例如“车辆后翼子板有补漆”、“更换了原厂轮胎”等，或强调某些特定车况。
 3.2 零售策略资料 (当选择“零售”时展示) 	-   3.2.1 填写处置渠道名称
-   字段说明：指定车辆将通过哪个具体的零售渠道进行销售。
-   功能说明：明确零售推广和销售的途径，便于渠道管理、效果追踪和资源分配。
-   交互说明：
-   展示方式：文本输入框。
-   必填：输入框后方带“*”标识。
-   字符限制：最少1个字符，最大50个字符。

 -   3.2.2 填写车辆类型
-   字段说明：在零售场景下，对车辆的特定业务分类。
-   功能说明：用于零售营销策略的区分，例如区分“精品车”、“经济适用车”、“特殊定制车”等，可能与批发类型有区别。
-   交互说明：
-   展示方式：文本输入框。
-   必填：输入框后方带“*”标识。
-   字符限制：最少1个字符，最大100个字符。



 -   3.2.3 公司标准保留价——同批发的标准保留价同一个信息
-   字段说明：同批发策略，根据公司政策，车辆最低可接受的出售价格。
-   功能说明：作为零售端议价和设定底价的内部参考基准，确保公司不亏损。
-   交互说明：
-   展示方式：文本展示，单位“万元”，精确到两位小数。
-   计算规则：同批发策略，由后台根据预设的计算公式和配置自动得出。
-   交互：不可编辑。

 -   3.2.4 填写寄售底价
-   字段说明：针对零售渠道，平台与车主或上家（若为寄售模式）约定好的车辆最低出售价格。
-   功能说明：该底价是零售销售的硬性门槛，任何低于此价格的成交都需要特殊审批。确保在零售过程中，不对车辆进行贱卖。
-   交互说明：
-   展示方式：数字输入框，单位“万元”，精确到两位小数。
-   必填：输入框后方带“”标识。
-   数据校验：
-   必须为正数，最多两位小数。
-   最大值9,999,999。
-   逻辑校验：寄售底价必须 ≥ 公司标准保留价。若低于，需填写说明
-   交互文案：提示“寄售底价，单位：万元”。

-   输入项2：保留价异常说明 (文本框)。
-   功能：用于补充说明为什么保留价低于公司标准保留价。
-   字符限制：最多200个字符。当低于公司标准保留价时必填。
-   显示文案：提示“请具体说明建议保留价低于公司保留价的理由”。
-   交互文案：提示“建议保留价，单位：万元”。
4. 策略备注 

	-   字段说明：对本次处置策略制定背景、特殊考量或后续跟进要点的补充说明。
-   功能说明：提供一个自由文本区域，方便运营人员记录策略的制定说明、特殊销售条件、市场判断等信息，供团队内部沟通、复盘和历史追溯。
-   交互说明：
-   展示方式：多行文本输入框。非必填
-   字符限制：最多500个字符。
-   交互文案：提示“备注策略的制定说明”。

 
5. 操作 	-   5.1 创建策略
-   字段说明：触发将所有填写信息提交至后端，生成新的处置策略记录。
-   功能说明：这是核心执行动作。成功创建后，车辆状态应由“待处置”更新为“执行中”，并进入后续的销售流程（如待发布拍卖或上架零售渠道）。
-   交互说明：
-   展示方式：主要操作按钮，通常位于弹窗底部居右侧。
-   按钮文案：“创建策略”。
-   前置校验：点击按钮时，前端会对所有必填项、数据格式、逻辑校验规则等进行再次验证。只有全部通过后，才能发送提交请求。
-   成功反馈：策略创建成功后，弹窗关闭，页面刷新或显示成功的提示信息（如“策略创建成功，车辆已进入执行中状态”）。“策略创建成功，车辆已进入‘执行中’状态，您可以前往处置策略列表查看或者修改。”
