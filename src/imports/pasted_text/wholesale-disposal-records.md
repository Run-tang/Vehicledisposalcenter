批发处置记录列表页需求详情
模块	功能说明
页面全局架构与导航	【功能说明】：
1. 业务入口：位于左侧目录树 [处置中心] -> 二级模块 [批发处置记录]。
2. Tab切换导航：包含 [全部]、[待处置]、[执行中]、[已完成]、[已取消] 五个状态维度。
【字段说明】：
- 标题+数量：例如“执行中 (12)”。数量来源：基于当前筛选器条件下，各状态对应的车辆策略总数。逻辑：实时刷新，点击不同Tab，下方列表展示对应状态的策略记录。
【交互说明】：点击Tab进行异步刷新列表，当前选中Tab高亮显示。
筛选分析区	【功能说明】：提供对海量批发记录的精准检索。包含 [重置] 与 [筛选] 操作。
【交互说明】：点击筛选实时更替列表数据；点击重置清空所有输入项并恢复至默认“全部”Tab。

1. 基础信息搜索 (车辆/标识)
【字段说明】：
1. 综合标识搜索：文本框。含义：支持策略ID、记录流水号、关联策略编号的精确或右模糊查询。来源：系统主键 ID。最大50字符
2. 车辆硬件搜索：文本框。含义：支持车牌号码、车架号（VIN）的后6位查询。来源：车辆档案。
3. 车辆品牌/车型：级联组件。含义：选择品牌后联动选车型。来源：资产库标准字典。
4. 所属门店：多选下拉。含义：筛选车辆最初收购的归属门店。来源：组织架构字典。
【交互说明】：支持回车键触发搜索，支持快速清空标签。

2. 业务状态筛选 (流转/渠道)
【字段说明】：
1. 车辆状态：多选下拉。枚举包含：【待拍卖（创建策略后默认）、检测待上拍、流拍、仲裁、成交】。来源：单次记录结果状态。
2. 处置渠道：多选下拉。枚举包含：[优加车拍-内部拍卖平台、优信、有辆、汽车街、线下、其他]。来源：处置渠道字典。
3. 车辆所在地：含义：筛选发拍时车辆所在的物理城市。来源：记录表物理位置字段。
4. 拍卖次数范围：数值区间。含义：输入最小值、最大值（最小0、最大99）。来源：汇总统计字段。逻辑：大于等于起值且小于等于止值。
【交互说明】：由于渠道较多，支持下拉框内通过关键词快速检索渠道名。

3. 财务与金额筛选 (价格/利润)
【字段说明】：
1. 收购价区间：金额区间（万元）。含义：根据收购价筛选。左值≤右值
2. 保留价区间：金额区间（万元）。含义：根据公司标准保留价筛选。来源：策略定价字段。左值≤右值
3. 最高出价/成交价区间：金额区间（万元）。含义：筛选高价或低价成交的记录。来源：记录表价格字段。左值≤右值
4. 进销价差状态/区间：复合组件。含义：① 筛选正/负偏差（盈利/亏损）；② 具体的价差金额范围。逻辑：[成交价] - [收购价]。支持负数，最小-9,999.99、最大9,999.99
5. 办证费区间：金额区间（元）。含义：筛选高额代办费记录。来源：手工回填字段。整数、左值≤右值
【交互说明】：金额区间需增加格式校验，左侧值不得大于右侧值，否则输入框变红提示。

4. 人员与时间维度 (溯源)
【字段说明】：
1. 中拍人/买家信息：文本框。含义：支持搜索中拍人姓名或联系电话。来源：买家档案或手工录入。
2. 中拍人注册地：级联组件。含义：针对内部平台车商，按其注册所在城市筛选。来源：车商画像库。
3. 操作人姓名：下拉搜索。含义：执行该笔更新记录的人员（定价师或库管）。来源：系统员工账户库。
4. 拍卖日期区间：日期组件（起止）。含义：该次拍卖动作发生的日期。
5. 成交时间区间：时间戳组件（起止）。含义：确认中拍的时点。来源：成交标志位记录的时间。
6. 更新时间区间：系统时间组件（起止）。含义：该条记录最后一次修改的时间。
【交互说明】：日期组件预设常用快捷项：[今日]、[本周]、[本月]、[最近三月]。
全局操作区（导出引擎）	【功能说明】：支持业务数据的离线分析。具备选中导出与全量导出两种策略。
【字段说明】：
1. 中拍人主要注册城市：来源：取内拍平台车商档案中的“收车偏好城市”。若外部回传或档案缺失展示“-”。
2. 成交车价：逻辑：内部拍卖取中拍确认价；外部拍卖取接口回传或人工录入的中拍金额。
3. 车辆佣金：逻辑：仅针对内部拍卖。金额 = 成交价 * 配置费率（由财务配置模块提供），单笔记录固定展示。外部渠道展示为“-”。
4. 所属门店：来源：映射收车时的原始门店全称——也是档案的场地名称。
【交互说明】：
- 导出部分：若前端选中复选框，则仅导出勾选行，交互无缝感知。
- 导出全部：若未勾选任何复选框，则导出当前筛选条件下的全量分页数据。导出文件格式为标准 .xlsx。
增加：过户资料回传进度：根据是否完成过户进行判断，枚举：未回传、已回传

列表字段详情（基础信息类）	【字段说明】：
1. 复选框：列表首列，用于批量导出。逻辑：支持全选及单选。
2. 记录流水号：系统生成唯一标识。含义：单次发拍动作的唯一ID。来源：处置记录记录表。
3. 关联策略编号：含义：该记录所属的处置策略主ID。来源：关联策略表。可点击跳转至策略详情页。
4. 车牌号码：展示所属车辆的车牌号。来源：车辆档案记录。
5. 车架号：展示17位VIN码。来源：车辆档案。
6. 品牌/车型：级联展示一级品牌+四级车型全称。来源：车辆档案同步。
7. 所属门店：含义：收车时责任归属的门店全称。来源：车辆档案映射。
【交互说明】：策略编号支持超链接交互，点击后在新标签页打开对应的策略详情页。
列表字段详情（财务与价格类）	【字段说明】：
1. 收购价：数值（万元）。来源：车辆采购合同录入金额。权限逻辑：敏感字段，根据角色控制可见性。
2. 公司标准保留价：数值（万元）。来源：创建&编辑策略时填写的价格快照。
3. 市场价格区间：文本展示（万元）。来源：创建&编辑策略时填写的参考区间快照。零售场景下展示"-"
4. 拍卖保留价：数值（万元）。含义：该次记录开拍时设定的起拍红线价。
5. 拍卖最高价：数值（万元）。含义：该场拍卖产生的最高有效标价。来源：系统自动抓取或人工回填。
6. 进销价差：数值（万元）。核心逻辑：[拍卖最高出价] - [收购价]。正负逻辑：若为负值（亏损），数值显示为红色。
7. 办证费：数值（元）。含义：拍卖成交后涉及的过户行政费用。来源：更新进度时手动录入。非必填。
【功能说明】：此模块用于监控单次发拍的盈亏状态，辅助定价师判定是否需要调整下一轮保留价。
六、列表字段详情（流转记录类）	【字段说明】：
1. 拍卖总次数：正整数。计算逻辑：针对同一车辆关联的所有已结束的处置记录行数进行合并计数（包含本次）。
2. 拍卖日期：显示执行拍卖的具体日期。来源：手动录入或外部回传。
3. 处置渠道：枚举值。来源：[优加车拍-内部拍卖平台、优信、有辆、线下]。逻辑：不同渠道决定了后续“操作”列的功能权限。
4. 车辆状态：枚举值。来源：【待拍卖（创建策略后默认）、检测待上拍、流拍、仲裁、成交】。根据拍卖结果动态更新。
5. 车辆所在地：文本。含义：发拍时车辆存放的实际物流城市。逻辑：默认映射库存地址，更新进度时若有修改则以此处为准。
6. 操作人姓名/更新时间：记录最后一次对此条记录进行编辑或确认的人员及系统时间戳。
7. 成交时间：显示中拍时刻。更新逻辑：若为内部拍，取内拍系统成交时间；若为外部拍，取人工录入成交进度的时刻。
8. 拍卖备注说明：长文本描述。来源：针对该次流拍原因或中拍异常的补充文字。
【交互说明】：点击“拍卖总次数”数字，可弹出简易气泡框展示历史所有频次的拍卖简要时间轴。
七、列表字段详情（买家信息类）	【字段说明】：
1. 中拍人姓名：文本。来源：成交后由系统抓取（内）或回填（外）。权限控制：普通岗位展示脱敏姓名（如：张*三）。
2. 中拍人电话：数值。来源：同姓名来源。仅在成交状态下显示。非必填。
3. 中拍人注册地信息：文本。映射逻辑：内部拍卖通过中拍人UID关联其注册时的“收车偏好”字段。外部中拍由于无法获取对方档案，固定展示为“-”。
4.过户资料回传进度：根据是否车辆完成过户进行判断，枚举：未回传、已回传。
【功能说明】：本模块用于画像分析，挖掘优质的分销车商聚集地。
八、操作逻辑模块	【功能说明】：根据不同处置渠道与交易结果，开放差异化操作路由。
【逻辑说明】：
1. 内部拍卖（优加车拍）：
- 未成交（流拍/未拍/）：开放 [查看出价详情]，展示该次拍卖的出价阶梯快照（含时间、出价人明细）。
- 有中拍（成交）：同时开放 [查看拍卖订单]（跳转拍卖订单详情页）及 [查看出价详情]。
2. 外部拍卖（优信/有辆等）：
- 未成交（仲裁/流拍等状态）：无需操作，保留静态记录。
- 有中拍：开放 [查看拍卖订单] 及 [回传拍卖资料]（跳转至资料附件上传页，补齐外部成交凭证）。
【交互说明】：点击“回传拍卖资料”时，系统应校验当前记录是否已由接口更新。若已有数据，则进入编辑模式；若无，则进入新增录入模式。

零售处置记录列表页需求详情
一、 零售处置记录列表页
处置中心 - 零售处置记录
模块	功能说明
 筛选器区域	【字段说明】：
1. 基础项：
记录流水号（最大30字符）
车架号（17位VIN码或后6位）
车牌号（支持模糊查询，最大10字符）
处置渠道（支持模糊查询。最大50字符）
「策略编号」筛选条件（文本框，最大30字符）
2. 分类项：
车辆类型（文本模糊搜索）、营运性质、所属门店。
3. 日期项：入库日期、上牌时间、订单时间（时间区间选择器）。
4. 财务项：
盈亏状态（枚举：盈利、亏损）、
零售价格区间（数值范围输入，万元，两位小数、最小0、最大999.99）。
寄售底价区间（数值范围输入，万元，两位小数、最小0、最大999.99）校验规则：左值≤右值
5. 摆放位置（文本模糊搜索，最大50字符）
【功能说明】：
- 提供对列表所有核心表头字段的筛选能力。所有筛选规则遵循 AND 逻辑。
【交互说明】：
1. 点击“筛选”按钮执行检索，点击“重置”清空所有已选/已输入的过滤条件并恢复默认展示。
2. 筛选器在滚动列表时建议吸顶。
操作区（导出）	【字段说明】：
1. 导出操作：按钮包含“导出部分”与“导出全部”。
【功能说明】：
- 导出部分：若用户在列表中勾选了复选框，点击导出仅产出勾选的数据行。
- 导出全部：若未勾选，系统按当前筛选条件（或全量）抓取数据生成报表。
- 字段补完：导出时需额外拼装非列表展示的底层数据（如车商注册城市、佣金计算、所属门店等）。导出列清单：【资产信息+业务与盈亏+额外字段（关联策略编号、公司标准保留价、市场参考价格区间、创建策略时间、创建策略人）】
【交互说明】：点击后无感处理，大批量数据在后台任务中心生成，完成后弹窗通知。

列表详情（资产信息）	【字段说明】：
1. 复选框：列表最左侧。用于多选操作。
2. 记录流水号：零售业务唯一识别码。来源：系统自动生成。更新逻辑：更新记录时生成，永不更改。
3. 收车日期：车辆进入零售库的时间。来源：库存模块。更新逻辑：随车辆收车单确认时间同步。
4. 关联策略编号
来源：处置策略表主键 ID
交互：可点击跳转至策略详情页（与批发列表保持一致）
5. 车架号/车牌号：车辆身份证及号牌。来源：车辆档案库。交互：点击可查看大图或复制文本。
6. 品牌/车型：车辆的一级品牌名称，车辆具体型号。来源：品牌库数据。要求：完整展示不缩写。
7. 身颜色/里程/上牌时间：物理属性。来源：收车评估快照数据。里程单位：km。
8. 车况说明：收车关键描述。来此创建零售策略时候的车况说明。
9. 车辆类型：来源：人工录入。
10. 原来所在/当前所在：物理具体存放城市/仓库。新零售记录时填写。
 列表详情（业务与盈亏）	默认排序：根据创建时间倒序排序。
【字段说明】：
1. 收购价：采购成本。来源：车辆档案。
2. 处置渠道：零售填写的渠道文本。
3. 摆放位置：具体展区/展位。
4. 寄售底价：策略设定的盈亏平衡参考价。
5. 零售价格：线下成交的车辆售价。
6. 状态：零售生命周期状态位（待处置、执行中、已完成等）。更新逻辑：业务操作流转自动触发。
7. 盈亏情况：
计算逻辑：盈亏 = 零售价格 - 收购价
成交价格来源：更新零售进度弹窗中勾选「是否成交=是」时填写的成交价格
未成交状态下盈亏列展示为"-
交互：结果为正（>=0）显示黑色数值；结果为负（<0）显示红色负数并带“-”号。
8.公司标准保留价（数值，万元）来源： 创建策略时 的公司标准保留价
9. 订单时间：首笔定金或成交合同时间。
10. 过户资料回传进度：根据售出后，是否完成过户进行判断，枚举：未回传、已回传。
操作	未成交：详情按钮
已成交：查看订单、详情
交互：点击[详情]跳转至对应的处置策略详情页。点击查看订单进入订单详情页。
